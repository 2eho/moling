| CSS 无性能优化 | 🟢 低 | 长列表滚动、动画无障碍 |

## 三、核心优化内容

### 3.1 React 性能优化

#### 3.1.1 React.memo 包裹纯展示组件

**问题**：所有组件每次父组件渲染时都重渲染，即使 props 没变化。

**修复**：用 `React.memo()` 包裹 12 个纯展示组件：

| 组件 | 文件 | 说明 |
|------|------|------|
| Button | `components/ui/Button.tsx` | 按钮组件，props: `children, variant, onClick` |
| Spinner | `components/ui/Spinner.tsx` | 加载指示器，无 props 变化 |
| Skeleton | `components/ui/Skeleton.tsx` | 骨架屏，无 props 变化 |
| Input | `components/ui/Input.tsx` | 输入框，props: `value, onChange` |
| Modal | `components/ui/Modal.tsx` | 模态框，props: `isOpen, onClose, children` |
| EmptyState | `components/ui/EmptyState.tsx` | 空状态，props: `icon, title, description` |
| Toast | `components/ui/Toast.tsx` | Toast 通知，props: `message, type` |
| AppShell | `components/layout/AppShell.tsx` | 布局外壳，props: `children` |
| Navbar | `components/layout/Navbar.tsx` | 导航栏，props: `user, onLogout` |
| SystemHealthBanner | `components/health/SystemHealthBanner.tsx` | 健康横幅，props: `status` |
| ProjectCard | `components/project/ProjectCard.tsx` | 项目卡片，props: `project, onClick` |
| ProjectStats | `components/project/ProjectStats.tsx` | 项目统计，props: `stats` |
| ProjectDetailModal | `components/project/ProjectDetailModal.tsx` | 项目详情弹窗，props: `project, isOpen, onClose` |

**注意**：`React.memo` 是浅比较。如果传递了每次渲染都新建的函数/对象，则 memo 失效。解决方案：
- 用 `useCallback` 稳定函数引用
- 用 `useMemo` 稳定对象引用
- 或将函数/对象放入 `useRef`（不触发重渲染）

#### 3.1.2 Context value 用 useMemo 稳定

**问题**：`createContext(defaultValue)` 的 value 每次渲染都是新对象，导致所有 Consumer 重渲染。

**修复**：在 4 个 Context Provider 中，用 `useMemo` 包裹 value：

```tsx
// ✅ 修复后：value 只有 deps 变化时才重建
const value = useMemo(() => ({
  user,
  isAuthenticated,
  login,
  logout,
  // ...
}), [user, isAuthenticated]);  // 只在这些依赖变化时重建

return <AuthContext.Provider value={value}>{children}</AuthContext.Provider>;
```

**修改文件**：
- `src/contexts/AuthContext.tsx`
- `src/contexts/WorkspaceContext.tsx`
- `src/contexts/ProjectContext.tsx`
- `src/contexts/SystemHealthContext.tsx`

#### 3.1.3 useRef 保存可变值

**问题**：`useState` 的值每次渲染都不同（触发依赖更新），但某些值不需要触发重渲染。

**修复**：在 `WorkspaceContext.tsx` 中：
- `generationParams` 从 `useState` 改为 `useRef`（不需要触发重渲染）
- `currentChapter` 用 `useRef` 保存（避免 effect 依赖整个对象）

```tsx
// ✅ 修复后：用 useRef 保存可变值
const generationParamsRef = useRef({
  mode: 'causal_chain',
  directions: [...],
  weights: {...},
});

// 读取时：generationParamsRef.current
// 修改时：generationParamsRef.current = {...}
// 不需要触发重渲染，所以不用 useState
```

### 3.2 useEffect 依赖修复

#### 3.2.1 RightPanel API 调用修复

**问题**：`RightPanel.tsx` 中有 `useEffect(() => { fetchCards() }, [currentChapter])`，
但 `currentChapter` 是对象，每次渲染都是新引用，导致 `fetchCards()` 在每次按键时都执行。

**修复**：只依赖对象的具体字段：

```tsx
// ❌ 修复前：对象引用作为依赖
useEffect(() => {
  if (currentChapter?.id) fetchCards();
}, [currentChapter]);  // 每次渲染都是新对象！

// ✅ 修复后：具体字段作为依赖
useEffect(() => {
  if (currentChapter?.id) fetchCards();
}, [currentChapter?.id, currentChapter?.project_id]);  // 只有 id 变化时才执行
```

#### 3.2.2 LandingPage IntersectionObserver 修复

**问题**：`LandingPage.tsx` 的 IntersectionObserver 回调中用了 `setVisibleElements(visibleElements + 1)`，
但 `visibleElements` 在依赖数组中，每次状态变化都触发 effect 重建（无限循环）。

**修复**：用函数式更新，并从依赖数组中移除 `visibleElements`：

```tsx
// ❌ 修复前：visibleElements 在依赖数组中
useEffect(() => {
  // ...
  setVisibleElements(visibleElements + 1);  // 依赖 visibleElements
  // ...
}, [visibleElements]);  // 每次变化都重建 Observer → 无限循环

// ✅ 修复后：函数式更新
useEffect(() => {
  // ...
  setVisibleElements(prev => prev + 1);  // 不依赖 visibleElements
  // ...
}, []);  // 空依赖数组 → 只创建一次 Observer
```

#### 3.2.3 workspace/page.tsx countdown 依赖补全

**问题**：`countdown` useEffect 缺少 `draftConfirmed` 和 `countdown` 依赖。

**修复**：补全依赖数组，并添加 `countdownValueRef` 避免闭包过期：

```tsx
useEffect(() => {
  if (!draftConfirmed) return;
  
  const timer = setInterval(() => {
    setCountdown(prev => {
      if (prev <= 1) {
        clearInterval(timer);
        return 0;
      }
      return prev - 1;
    });
  }, 1000);
  
  return () => clearInterval(timer);
}, [draftConfirmed]);  // ✅ 添加 draftConfirmed 依赖
```

### 3.3 列表 key 修复

**问题**：列表渲染使用 `key={idx}`（数组索引作为 key），当列表顺序变化时，
React 无法正确识别哪些元素变了，导致状态混乱或性能下降。

**修复**：用唯一标识作为 key：

| 文件 | 修复前 | 修复后 |
|------|--------|--------|
| `WorldList.tsx` | `key={idx}` (rules map) | `key={`${rule}-${idx}`}` |
| `WorldList.tsx` | `key={idx}` (factions map) | `key={f.name || \`faction-${idx}\`}` |
| `CharacterList.tsx` | `key={i}` (relationships map) | `key={rel.character_id || \`rel-${i}\`}` |
| `TimelineList.tsx` | `key={idx}` (events map) | `key={`ch${event.chapter_number}-${idx}`}` |

**何时可以用索引作为 key**：
- 列表是静态的（永远不会重新排序或过滤）
- 列表项没有状态（如输入框的内容）
- 你确定列表顺序永远不变

### 3.5 CSS 性能优化

**文件**：`moling-web/src/app/globals.css`

| 优化项 | CSS 属性 | 说明 |
|---------|-----------|------|
| 跳过视口外渲染 | `content-visibility: auto` | 跳过视口外元素的渲染（提升滚动性能） |

| 动画无障碍 | `@media (prefers-reduced-motion: reduce)` | 允许用户禁用动画 |
| 对比度无障碍 | `@media (prefers-contrast: more)` | 高对比度模式 |
| 焦点样式 | `:focus-visible` | 只用键盘导航时显示焦点样式 |
| Skip link | `.skip-link` | 键盘导航跳过导航栏 |

**使用示例**：

### 3.6 API 客户端优化

**文件**：`moling-web/src/lib/apiClient.ts`

| 优化项 | 实现 | 说明 |
|---------|------|------|
| 请求去重 | `inflightRequests: Map<string, Promise>` | 同一 URL 的并发请求合并为一个 |
| 响应缓存 | `responseCache: Map<string, {data, expiry}>` | GET 请求缓存 5 秒，最多缓存 50 条 |
| Token 刷新去重 | `refreshPromise: Promise \| null` | 多个 401 只触发一次 token 刷新 |
| 导出清除缓存 | `export function clearApiCache()` | 提供手动清除缓存的接口 |

**请求去重示例**：

```typescript
const inflightRequests = new Map<string, Promise<any>>();

async function deduplicatedFetch(url: string, options: RequestInit): Promise<any> {
  const key = `${url}:${JSON.stringify(options.body)}`;
  
  if (inflightRequests.has(key)) {
    return inflightRequests.get(key)!;  // 复用进行中的请求
  }
  
  const promise = fetch(url, options).then(res => res.json());
  inflightRequests.set(key, promise);
  
  // 请求完成后从 Map 中移除
  promise.finally(() => inflightRequests.delete(key));
  
  return promise;
}
```

### 3.7 动态导入重型组件

**文件**：`moling-web/src/app/workspace/[projectId]/page.tsx`

用 `next/dynamic` 将重型组件改为动态导入（减少首屏 JS 体积）：

| 组件 | 动态导入 | 说明 |
|------|--------|------|
| LeftPanel | `dynamic(() => import('@/components/workspace/LeftPanel'))` | 左栏面板，首屏不需要 |
| Editor | `dynamic(() => import('@/components/workspace/Editor'))` | 编辑器，首屏不需要 |
| RightPanel | `dynamic(() => import('@/components/workspace/RightPanel'))` | 右栏面板，首屏不需要 |
| CardModal | `dynamic(() => import('@/components/workspace/CardModal'))` | 抽卡模态框，稀有使用 |
| GenerationProgress | `dynamic(() => import('@/components/workspace/GenerationProgress'))` | 生成进度，稀有使用 |

**注意**：
- 动态导入会增加网络请求数（每个 `dynamic()` 一个单独的文件）
- 不要对小型组件（< 5KB）使用动态导入
- 用 `loading` 属性提供加载状态，避免布局偏移

### 8.2 关键经验

1. **React.memo 是浅比较**：如果传递了每次渲染都新建的函数/对象，则 memo 失效
2. **Context value 必须用 useMemo 稳定**：否则所有 Consumer 都会重渲染
3. **useEffect 依赖数组 NEVER 放对象/数组引用**：用具体字段或函数式更新
4. **列表 key 必须用唯一标识**：不能用索引（除非列表永远不变）
5. **API 客户端要优化**：请求去重、响应缓存、Token 刷新去重
6. **动态导入减少首屏体积**：对重型组件用 `next/dynamic`
7. **CSS 也能优化性能**：`content-visibility: auto` 跳过视口外渲染
