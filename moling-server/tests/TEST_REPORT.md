## 一、测试概述

### 1.1 测试范围

本次测试覆盖墨灵项目后端 API 的所有已实现端点，包括：

- 认证 API（注册、登录、刷新 Token、获取当前用户）
- 项目 API（CRUD 操作）
- 章节 API（CRUD 操作、重新排序）
- 卡牌 API（列表、抽卡、创建、退役、历史）
- 四库 API（人物、时间线、剧情承诺、世界观的 CRUD）
- 生成 API（开始生成、任务状态、取消任务）
- 通知 API（列表、未读计数、标记已读、全部已读、删除）
- 设置 API（获取、更新、修改密码、个人资料）
- 秘密 API（列表、按角色查询、更新）

### 1.2 测试方法

- **测试框架**：Pytest + HTTPX（异步测试客户端）
- **测试模式**：AAA（Arrange-Act-Assert）
- **数据库**：SQLite 内存数据库（测试隔离）
- **认证**：JWT Token（通过固件自动处理）

### 1.3 测试类型

| 测试类型 | 数量 | 说明 |
|---------|------|------|
| 正向测试 | 30+ | 验证 API 在正常输入下的正确响应 |
| 反向测试 | 20+ | 验证 API 在错误输入下的错误处理 |
| 集成测试 | 4 | 测试完整的用户工作流 |

## 四、已知问题和限制

### 4.1 Windows 兼容性问题

**问题**：在 Windows 上运行测试时，由于 `greenlet` DLL 缺失，所有需要数据库的测试会被自动跳过。

**影响**：无法在 Windows 上直接运行测试。

**解决方案**：
1. 在 Linux/macOS 上运行测试
2. 使用 CI/CD 管道运行测试
3. 修改代码以支持同步数据库（仅用于测试）

### 4.2 未实现的 API 端点

根据 `backend_gap_analysis.md`，以下 API 端点尚未实现，因此无法测试：

**P0（必须实现）**：
- `POST /api/v1/projects/:pid/chapters/:id/confirm`（确认收纳）
- `POST /api/v1/projects/:pid/chapters/revise`（拒稿/修订）
- 四库的单个 item 更新/删除端点（部分缺失）

**P1（重要）**：
- 密码重置 API
- 健康监控设置 API

**P2（次要）**：
- 管理后台 API
- 导入引擎 API
- 编织模式 API

### 4.3 测试限制

1. **LLM 服务依赖**：生成 API 测试需要 LLM 服务配置，如果未配置可能返回 500/503
2. **时间依赖**：Token 过期测试需要时间模拟
3. **并发测试**：未测试并发请求场景
4. **性能测试**：未测试 API 响应时间

## 五、测试执行说明

### 5.1 运行测试（Linux/macOS）

```bash
cd moling-server
source .venv/bin/activate
pytest tests/test_api/ -v
```

### 5.2 运行测试（Windows）

由于 Windows 上 greenlet DLL 缺失，测试会被自动跳过。建议：

1. 使用 WSL2（Windows Subsystem for Linux）
2. 使用 Docker 容器
3. 在 CI/CD 管道中运行

### 5.3 运行特定测试文件

```bash
# 运行认证 API 测试
pytest tests/test_api/test_auth_api.py -v

# 运行集成测试
pytest tests/test_api/test_integration.py -v

# 运行特定测试类
pytest tests/test_api/test_auth_api.py::TestAuthLogin -v

# 运行特定测试方法
pytest tests/test_api/test_auth_api.py::TestAuthLogin::test_login_success -v
```

## 六、后续工作建议

### 6.1 优先级 P0

1. **补充缺失的 API 端点测试**：一旦后端实现了缺失的端点，立即补充测试
2. **增强权限控制测试**：添加跨用户访问测试，验证用户只能访问自己的数据
3. **添加边界条件测试**：测试分页边界、字段长度限制等

### 6.2 优先级 P1

1. **创建前端组件测试**：使用 Jest + React Testing Library 测试关键组件
2. **创建端到端测试**：使用 Playwright 或 Cypress 测试完整的用户流程
3. **添加性能测试**：测试 API 响应时间和并发处理能力

### 6.3 优先级 P2

1. **添加安全测试**：测试 SQL 注入、XSS、CSRF 等安全漏洞
2. **添加兼容性测试**：测试不同浏览器、设备的兼容性
3. **添加回归测试**：在修复 Bug 后添加回归测试
