## 四、发现的问题

### 1. 速率限制过于严格

后端启用了速率限制中间件（`RateLimitMiddleware`），配置为：
- 100 请求/60秒

这导致性能测试时大量请求被拒绝（429）。

**建议**：
- 性能测试时临时提高速率限制阈值
- 或为性能测试创建专用配置

### 2. 部分端点返回 500 错误

以下端点存在问题：
- `POST /api/v1/auth/login` - 可能缺少依赖或配置

**建议**：
- 检查认证服务实现
- 验证数据库连接和 Redis 连接

### 3. Locust 在 Windows 上不兼容

使用 Python 3.13 + Locust 2.44.3 时遇到 gevent/greenlet DLL 加载错误。

**建议**：
- 使用 WSL2 运行 Locust
- 或使用已创建的 simple_perf_test.py 脚本

## 五、性能评估

| 指标 | 目标 | 实际 | 是否达标 |
|:-----|:-----|:-----|:---------|
| **P95 响应时间** | < 500ms | 234ms (部分成功) | ⚠️ 需优化 |
| **成功率** | > 95% | 64% (极简测试) | ❌ 不达标 |
| **吞吐量** | > 10 req/s | 15.6 req/s | ✅ 达标 |

**结论**：
- 后端基础功能可用，但稳定性需改进
- 速率限制影响了性能测试的准确性
- 建议修复 500 错误端点后重新测试
