## 性能分析

### 关键问题

1. **成功率极低** (18% - 35%)
   - 简单测试成功率仅18%
   - 速率限制测试成功率仅35%
   - **根本原因**: 大部分请求失败，可能原因：
     - 认证token无效或过期
     - 数据库约束错误（如唯一性约束）
     - 后端业务逻辑错误

2. **P95响应时间超标** (3492.44 ms)
   - 目标: P95 < 200ms
   - 实际: P95 = 3492.44ms (简单测试)
   - **不达标**: 超出目标17倍
   - 速率限制测试的P95较好 (14.6ms)，但成功率低

3. **吞吐量未达标**
   - 认证API目标: RPS > 50
   - 实际吞吐量: 27.24 请求/秒 (简单测试), 7.1 请求/秒 (速率限制测试)
   - **不达标**: 未达到目标

### 端点性能对比

**性能较好的端点**:
- `GET /api/v1/projects`: 平均 7.64ms, P95 9.38ms
- 这些端点响应快，但成功率未知

**性能较差的端点**:
- `POST /api/v1/projects`: 平均 394.94ms, P95 3493.76ms

- **注意**: 高响应时间可能是因为请求失败后的超时

## 问题排查建议

### 1. 认证问题
- **现象**: 大量请求失败
- **排查**:
  - 检查测试脚本中的token是否正确
  - 确认测试用户是否已创建
  - 检查认证中间件的逻辑

### 2. 数据库问题
- **现象**: POST请求（创建项目）失败率高
- **排查**:
  - 检查数据库约束（如唯一性约束）
  - 查看后端日志中的SQL错误
  - 确认测试数据是否冲突

### 3. 超时问题
- **现象**: P95响应时间极高（约3.5秒）
- **排查**:
  - 检查后端是否有死锁或长查询
  - 确认数据库连接池配置
  - 检查Redis连接是否正常

### 4. 编码问题
- **现象**: Windows控制台Unicode错误
- **解决**: 设置环境变量 `PYTHONIOENCODING=utf-8`
