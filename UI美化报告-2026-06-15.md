## 一、优化背景

### 现状问题
1. **设计令牌不统一**：`globals.css` 与组件 CSS 使用不同的变量命名系统
   - `globals.css` 使用 `--space-2`、`--color-brand-indigo`
   - 组件 CSS 使用 `--spacing-2`、`--color-brand-primary`
   - 导致样式变量冲突，视觉效果不一致

2. **视觉精致度不足**：
   - 圆角偏小（4/8/12px），不够现代
   - 缺乏阴影层次，组件扁平
   - 交互反馈不足（无 hover/focus 动效）
   - 过渡生硬，缺乏流畅性

3. **组件一致性差**：
   - 不同页面/组件使用不同的圆角/间距/颜色
   - 重复实现类似的样式逻辑
   - 缺乏统一的设计语言

| **阴影系统** | `--shadow-card` | `0 2px 8px rgba(0,0,0,0.15)` | 卡片阴影 |
| | `--shadow-elevated` | `0 8px 24px rgba(0,0,0,0.25)` | 浮起阴影 |
| | `--shadow-modal` | `0 16px 48px rgba(0,0,0,0.4), 0 8px 16px rgba(0,0,0,0.2)` | 模态框阴影 |
| | `--shadow-glow-indigo` | `0 0 20px rgba(99, 102, 241, 0.3)` | 品牌色光晕 |
| | `--shadow-glow-amber` | `0 0 20px rgba(212, 168, 67, 0.3)` | 琥珀色光晕 |
| **过渡系统** | `--transition-fast` | `100ms cubic-bezier(0.4, 0, 0.2, 1)` | 快速过渡 |

1. **圆角微调**：所有圆角增大 2px，更现代柔和
2. **变量兼容**：保留旧变量名作为别名，确保旧组件不报错
3. **阴影层次**：新增 5 级阴影，覆盖卡片/浮起/模态框/光晕等场景
4. **过渡统一**：所有交互元素使用 `var(--transition-normal)`（200ms ease-out）
