- 该方法包含核心生成逻辑（之前在 Celery 任务中）
- 修改 `app/worker/tasks.py` 中的 `run_generation_task()`，使其调用 service 方法
- 使 Celery 任务成为 service 层的薄包装器

**修改前** (tasks.py):
```python
async def _execute_generation(task_id: str, celery_task) -> None:
    # 直接调用 DAO 和 LLM
    from app.dao import generation_dao
    from app.llm import llm_client
    ...
```

**修改后** (tasks.py):
```python
@celery_app.task(bind=True, max_retries=2, default_retry_delay=30)
def run_generation_task(self, generation_task_id: str) -> dict:
    service = GenerationService()
    result = asyncio.run(service.execute_generation(None, generation_task_id))
    return result
```

**影响文件**:
- `app/service/generation_service.py`
- `app/worker/tasks.py`
