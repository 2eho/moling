## 关键问题汇总

### 🔴 高优先级（核心功能缺失）

1. **API 对接全部缺失** - 所有 Tab 均使用硬编码 mock 数据，无实际 API 调用
2. **用户管理：新建/编辑/封禁/删除功能缺失** - 仅有 UI 按钮，无对应功能实现
3. **支付记录：退款功能缺失** - 文档明确要求，代码中无实现
4. **LLM 用量：模型用量分布图缺失** - 文档明确要求，代码中完全未实现
5. **搜索防抖未实现** - 文档明确要求 300ms 防抖，代码中无实现

### 🟡 中优先级（功能不完整）

6. **表格排序未实现** - 有 UI 但无排序逻辑
7. **分页无实际功能** - UI 完整但无真实分页逻辑
8. **批量操作栏缺失** - 多选 UI 已实现，但无批量操作栏
9. **API Key 脱敏显示未实现** - 加载后应如何显示脱敏 Key 的逻辑缺失
10. **空状态/加载状态/错误状态未处理** - 所有 Tab 均无这三种状态处理

### 🟢 低优先级（体验优化）

11. **LLM 用量 StatCard 与文档不一致** - 显示的数据项与文档定义不同
12. **响应式：移动端抽屉式侧边栏未实现** - 小屏时侧边栏缩小但未变为 drawer overlay
13. **日期筛选与文档不一致** - LLM 用量页的日期选项与文档不同
14. **保存成功提示不完整** - 应显示"将在下次 LLM 调用时生效"

## 代码质量备注

1. **Mock 数据硬编码** - `users`、`payments`、`activities` 数组直接写在组件内，应使用 API 获取数据
2. **全选功能仅硬编码5行** - `toggleSelectAll` 中硬编码了 `[0,1,2,3,4]`，应基于实际数据动态处理
3. **无错误处理** - 所有 fetch 请求缺乏完善的错误处理
4. **无加载状态** - 数据请求期间无 Loading 指示器
5. **LLM 配置测试前自动保存** - `handleTestConnection` 中先执行保存再测试，与文档描述的流程有差异
