## 三、源码中的硬编码配置

> **重要**：为避免 `.env` 被 `.gitignore` 忽略导致部署失败，关键配置已硬编码到源码中。

### 3.1 前端 API 地址

**文件**：`moling-web/src/lib/apiClient.ts`

```typescript
function buildUrl(path: string): string {
  const baseUrl = process.env.NEXT_PUBLIC_API_BASE_URL || '';

  // 1. 空 baseUrl（开发环境）→ 使用相对路径
  if (!baseUrl) {
    return path;
  }

  // 2. 相对路径前缀（如 /moling/api/v1）→ 直接拼接
  if (baseUrl.startsWith('/')) {
    return `${baseUrl}${path}`;
  }

  // 3. 完整 URL → 拼接
  return `${baseUrl}${path}`;
}
```

**配置文件** `.env.production`：

```
NEXT_PUBLIC_API_BASE_URL=/moling/api/v1
```

**构建时注入**（通过 `Dockerfile`）：

```dockerfile
ARG NEXT_PUBLIC_API_BASE_URL
ENV NEXT_PUBLIC_API_BASE_URL=${NEXT_PUBLIC_API_BASE_URL}
```

### 3.2 后端 CORS 配置

**文件**：`moling-server/app/main.py`（或 `config.py`）

```python
# 默认值包含 localhost（开发）和生产环境 origin
# 生产环境通过环境变量 CORS_ORIGINS 覆盖
CORS_ORIGINS = os.getenv(
    "CORS_ORIGINS",
    "http://localhost:8080,http://localhost:3000"
).split(",")
```

**通过环境变量覆盖**（生产环境）：

```yaml
# docker-compose.yml
environment:
  # 替换为实际服务器 IP
  - CORS_ORIGINS=http://服务器IP:8080
```

### 3.3 数据库 URL

**文件**：`moling-server/docker-compose.yml`

```yaml
environment:
  - DATABASE_URL=postgresql+asyncpg://moling:moling@db:5432/moling
```

> 容器内通过服务名 `db` 访问 PostgreSQL，不走 `localhost`。

## 四、Nginx 配置

**源码位置**：`deploy/nginx/moling.conf`

**部署时复制到**：`/etc/nginx/conf.d/moling.conf`

```nginx
server {
    listen 8080;
    listen [::]:8080;
    server_name _;

    # 根路径 → 重定向到 /moling
    location = / {
        return 301 /moling;
    }

    # 前端 → Next.js (3000)
    location /moling {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # API → FastAPI (8000)
    location /moling/api/ {
        rewrite ^/moling(/api/.*)$ $1 break;
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_connect_timeout 60s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
        proxy_buffering off;
    }
}
```

**关键逻辑**：

```
浏览器请求：http://服务器IP:8080/moling/api/v1/auth/login
  → Nginx 匹配 location /moling/api/
  → rewrite 去掉 /moling 前缀
  → 转发到 http://127.0.0.1:8000/api/v1/auth/login
```

## 五、部署流程

### 5.1 首次部署

```bash
# 1. 克隆代码
cd /root
git clone git@github.com:2eho/moling.git
cd moling

# 2. 复制 Nginx 配置
sudo cp deploy/nginx/moling.conf /etc/nginx/conf.d/
sudo nginx -t          # 测试配置
sudo nginx -s reload   # 重载 Nginx

# 3. 构建并启动所有服务
docker compose up -d --build

# 4. 验证
curl http://localhost:8000/api/v1/health
# 预期：{"status":"healthy","version":"0.1.0","service":"moling-api"}

curl http://localhost:8080/moling
# 预期：返回 HTML 页面
```

### 5.2 更新部署

```bash
# 1. 拉取最新代码
cd /root/moling
git pull origin main

# 2. 如果 Nginx 配置有更新
sudo cp deploy/nginx/moling.conf /etc/nginx/conf.d/
sudo nginx -s reload

# 3. 重新构建并启动
docker compose up -d --build

# 4. 查看日志，确认无错误
docker compose logs app --tail=50
docker compose logs web --tail=50
```

## 七、常见问题排查

### 7.1 `Failed to fetch`

**原因**：前端 API 地址配置错误。

**排查**：

```bash
# 1. 检查前端构建时的环境变量
docker compose exec web printenv | grep NEXT_PUBLIC_API_BASE_URL

# 2. 检查浏览器 Network 标签，看请求发到哪里
# 正确：/moling/api/v1/auth/login（相对路径）
# 错误：http://localhost:8000/api/v1/...（绝对路径）
```

**修复**：

- 确认 `apiClient.ts` 中的 fallback 地址正确
- 确认 `Dockerfile` 中 `ARG NEXT_PUBLIC_API_BASE_URL` 默认值正确
- 重新构建：`docker compose up -d --build web`

### 7.2 404 Not Found（API 请求）

**原因**：Nginx 未正确反代 `/moling/api/`。

**排查**：

```bash
# 1. 检查 Nginx 配置是否存在
cat /etc/nginx/conf.d/moling.conf

# 2. 测试 Nginx 配置
sudo nginx -t

# 3. 直接访问后端（绕过 Nginx）
curl http://localhost:8000/api/v1/health

# 4. 通过 Nginx 访问
curl http://localhost:8080/moling/api/v1/health
```

**修复**：

- 确认 `deploy/nginx/moling.conf` 已复制到 `/etc/nginx/conf.d/`
- 确认 `rewrite ^/moling(/api/.*)$ $1 break;` 正确
- 重新加载 Nginx：`sudo nginx -s reload`

### 7.3 后端启动失败（PostgreSQL 连接拒绝）

**原因**：PostgreSQL 容器未启动，或 `DATABASE_URL` 配置错误。

**排查**：

```bash
# 1. 检查容器状态
docker compose ps

# 2. 检查 db 容器日志
docker compose logs db

# 3. 检查后端环境变量
docker compose exec app printenv | grep DATABASE_URL
```

**修复**：

- 确认 `docker-compose.yml` 中有 `db` 服务
- 确认 `app` 服务的 `depends_on` 包含 `db`
- 确认 `DATABASE_URL` 使用服务名 `db` 而不是 `localhost`

### 7.4 注册/登录 404 或 422

**原因**：前端字段名与后端不匹配。

**已修复**：

| 问题 | 修复 |
|:---|:---|
| 前端发 `username`，后端收 `nickname` | `AuthContext.tsx` 注册字段改为 `nickname` |
| 重置密码路径错误 | `/auth/reset-password` → `/auth/password-reset-request` |
