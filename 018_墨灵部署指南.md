# 墨灵 (Moling) 生产部署指南

> 适用场景：Nginx 反向代理 + 前后端同机部署

---

## 一、把新构建部署到服务器

### 方式 A：手动覆盖（最简单）

在**本机**把构建好的文件打包：

```bash
# 在本机执行（Windows）
cd C:\Users\Admin\Desktop\MolingProject\moling-web
# .next 目录已构建完成，直接打包
tar -czf moling-next.tar.gz .next
```

把包传到服务器，解压覆盖：

```bash
# 在服务器上执行（Linux）
cd /path/to/moling-web
# 备份旧的（以防万一）
mv .next .next.bak.$(date +%Y%m%d)
# 解压新的
tar -xzf ~/moling-next.tar.gz
```

### 方式 B：rsync（推荐，按需增量同步）

```bash
# 在服务器上执行
rsync -avz --exclude 'node_modules' \
  user@你的本机IP:/path/to/moling-web/.next \
  /path/to/moling-web/
```

### 方式 C：git pull（如果服务器上也是 git 项目）

```bash
# 在服务器上执行
cd /path/to/moling-web
git pull origin main
npm run build   # 重新构建
```

## 二、重启前端服务

根据你的部署方式，选择一种：

### 用 PM2

```bash
pm2 restart moling-web
pm2 logs moling-web --lines 20   # 查看日志，确认启动成功
```

### 用 systemd

```bash
sudo systemctl restart moling-web
sudo systemctl status moling-web   # 确认状态是 active (running)
```

### 用 Docker

```bash
docker-compose restart web
docker-compose logs web --tail=20
```

## 三、确认后端服务正常

```bash
# 后端在服务器上应该运行在 localhost:8080
curl http://localhost:8080/api/v1/health
```

预期返回：
```json
{"status":"healthy","version":"0.1.0","service":"moling-api"}
```

如果后端没启动：

```bash
# 进入后端目录
cd /path/to/moling-server
# 激活虚拟环境并启动
source venv/bin/activate
nohup uvicorn app.main:app --host 0.0.0.0 --port 8080 &
```

## 四、确认 Nginx 配置正确

### 4.1 检查 nginx 配置文件

```bash
# 查看 nginx 配置
cat /etc/nginx/sites-available/default
# 或者
cat /etc/nginx/conf.d/default.conf
```

### 4.2 关键配置

确保文件中有以下内容：

```nginx
server {
    listen 80;
    server_name _;  # 或者你的域名

    # 前端静态文件
    root /path/to/moling-web;
    index index.html;

    # API 反向代理（重点！）
    location /api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # Swagger 文档
    location /docs {
        proxy_pass http://localhost:8080/docs;
    }

    location /openapi.json {
        proxy_pass http://localhost:8080/openapi.json;
    }

    # Next.js 动态请求
    location /_next/ {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
    }

    # 其他请求转发到 Next.js
    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

> ⚠️ **`location /api/` 那一段必须有！** 否则前端请求 `/api/v1/auth/login` 会 404。

### 4.3 重启 nginx

```bash
# 测试配置
sudo nginx -t

# 如果提示 syntax is ok，说明配置没问题，重启
sudo nginx -s reload
```

## 五、部署后验收

### 5.1 检查前端能否访问

打开浏览器，访问你的服务器地址：

```
http://你的服务器IP
```

页面应该正常显示。

### 5.2 实际操作测试

1. 打开 F12 开发者工具 → **Network** 标签
2. **点击"登录"按钮**，输入任意内容提交
3. 看 Network 里是否有请求发出

#### ✅ 成功标志

```
Request URL: /api/v1/auth/login    ← 相对路径，说明配置正确
Request Method: POST
Status Code: 401                   ← 401 是正常的（测试账号不存在）
                                   ← 401 说明后端成功收到了请求！
```

注意：哪怕后端返回 401/400，只要不是 `failed to fetch` 就说明**前后端通了**。

#### ❌ 失败标志

```
Request URL: http://YOUR_SERVER_IP:8000/api/v1/auth/login  ← 绝对路径指向错误地址
Status: (failed) net::ERR_NAME_NOT_RESOLVED                ← 域名解析失败
或
Status: (failed) net::ERR_CONNECTION_REFUSED               ← 连接被拒绝
或
Status: (failed) TypeError: Failed to fetch                ← 典型的连不上
```

### 5.3 验证 nginx 确实代理了 API

```bash
# 在服务器上模拟前端发请求
curl http://localhost/api/v1/health
```

预期返回：
```json
{"status":"healthy","version":"0.1.0","service":"moling-api"}
```

如果能拿到，说明 nginx → 后端 通了。

## 六、常见问题

| 问题 | 原因 | 解决 |
|:---|:---|:---|
| 页面白屏/404 | `.next` 没部署或路径不对 | 确认 `.next` 目录已覆盖到服务器正确位置 |
| `failed to fetch` | API请求地址不对或nginx没配`/api/`代理 | 检查 nginx 配置中的 `location /api/` 段 |
| 登录返回 401 | 正常（测试账号不存在） | 先去注册页面注册 |
| 登录返回 500 | 后端数据库未初始化 | 进后端目录执行数据库迁移 |
| 图片/样式加载不出来 | `.next/static` 路径不对 | 确认静态文件路径正确 |

```
部署步骤 = 上传 .next → 重启前端 → 检查 nginx /api/ 代理 → 打开浏览器按F12验证
```
