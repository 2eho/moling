## 2. 前端代码结构 (moling-web)

### 2.1 架构总体评价

| 维度 | 评分 | 说明 |
|------|:----:|------|
| 路由结构 | 🟢 8/10 | Next.js App Router 组织良好 |
| 组件设计 | 🟢 8/10 | 组件拆分合理，粒度适中 |
| 状态管理 | 🟡 7/10 | Context 模式使用 `useMemo` 稳定 value，但存在重复状态 |
| API 层 | 🟡 6/10 | 两套 API 层并存，活动层 968 行过于臃肿 |
| 类型安全 | 🔴 4/10 | `ignoreBuildErrors: true` + 死代码类型文件 |
| Mock 系统 | 🟡 6/10 | 覆盖全面但路径与真实 API 不匹配 |
| 测试覆盖 | 🟡 6/10 | 有 Vitest 测试 (33 passing) |

### 2.2 严重问题 (S级)

#### S7 - TypeScript 和 ESLint 被完全跳过
- **位置**: `next.config.ts:10-19`
- **影响**: `ignoreBuildErrors: true` + `ignoreDuringBuilds: true`，任何类型错误和 lint 警告都不会阻止构建，运行时 bug 风险高
- **修复建议**: 逐步修复类型问题后移除这两个配置

#### S8 - 废弃 API 层死代码 (src/api/)
- **位置**: `src/api/client.ts`, `types.ts`, `index.ts`, `workspace.ts`, `vault.ts`, `settings.ts`, `notifications.ts`, `import.ts`
- **影响**: 8 个文件 ~500 行死代码，与活动层类型不一致（驼峰 vs 蛇形命名），token key 不同
- **修复建议**: 确认无引用后删除整个目录

#### S9 - Mock API 路径与真实 API 不匹配
- **位置**: `src/mock/index.ts:289,304`
- **影响**: Mock 注册的生成 API 路径（`/generation/trigger`）与真实路径（`/generate/chapters/...`）不同，Mock 模式下生成流程无法工作
- **修复建议**: 统一 Mock 路径与真实路径

#### S10 - 两套 Landing Page 实现
- **位置**: `src/app/landing/page.tsx` + `src/components/landing/LandingPage.tsx`
- **影响**: 两份独立代码，`src/app/page.tsx` 导入组件版，`src/app/landing/page.tsx` 使用页面版。存在 `.bak` 备份文件
- **修复建议**: 删除冗余版本
