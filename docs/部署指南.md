## 方案1：Docker部署（推荐）

### 1.1 创建Dockerfile（后端）

**文件：`moling-server/Dockerfile`**
```dockerfile
FROM python:3.12-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    postgresql-client \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令（使用Gunicorn）
CMD ["gunicorn", "app.main:app", "-w", "4", "-k", "uvicorn.workers.UvicornWorker", "--bind", "0.0.0.0:8000"]
```

### 1.2 创建Dockerfile（前端）

**文件：`moling-web/Dockerfile`**
```dockerfile
# 构建阶段
FROM node:22-alpine AS builder

WORKDIR /app

# 复制依赖文件
COPY package.json pnpm-lock.yaml ./

# 安装依赖
RUN npm install -g pnpm && pnpm install

# 复制源代码
COPY . .

# 构建生产版本
RUN pnpm run build

# 生产阶段
FROM node:22-alpine

WORKDIR /app

# 复制构建产物
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/public ./public
COPY --from=builder /app/package.json ./
COPY --from=builder /app/next.config.js ./
COPY --from=builder /app/node_modules/.pnpm ./node_modules/.pnpm

# 安装生产依赖
RUN npm install -g pnpm && pnpm install --prod

# 暴露端口
EXPOSE 3000

# 启动命令
CMD ["pnpm", "start"]
```

### 1.3 创建docker-compose.yml

**文件：`docker-compose.yml`**
```yaml
version: '3.8'

services:
  # 后端服务
  backend:
    build:
      context: ./moling-server
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://moling:password@db:5432/moling
      - REDIS_URL=redis://redis:6379/0
      - ENVIRONMENT=production
      - LLM_API_KEY=${LLM_API_KEY}
      - LLM_API_BASE=${LLM_API_BASE}
    depends_on:
      - db
      - redis
    restart: unless-stopped

  # 前端服务
  frontend:
    build:
      context: ./moling-web
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NEXT_PUBLIC_API_URL=http://localhost:8000/api/v1
    depends_on:
      - backend
    restart: unless-stopped

  # 数据库
  db:
    image: postgres:16-alpine
    environment:
      - POSTGRES_USER=moling
      - POSTGRES_PASSWORD=password
      - POSTGRES_DB=moling
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    restart: unless-stopped

  # Redis缓存
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    restart: unless-stopped

  # Nginx反向代理（可选）
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - frontend
      - backend
    restart: unless-stopped

volumes:
  postgres_data:
```

### 1.4 启动Docker服务

```bash
# 构建并启动所有服务
docker-compose up -d --build

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f backend
docker-compose logs -f frontend

# 停止服务
docker-compose down
```

## 方案2：CloudBase部署（腾讯云）

### 2.1 后端部署（CloudBase）

1. **登录CloudBase控制台**
   - 访问 https://console.cloud.tencent.com/tcb
   - 创建新环境（选择杭州/上海区域）

2. **创建云托管服务**
   - 进入"云托管" -> "服务列表"
   - 点击"新建服务"，上传`moling-server`代码
   - 配置：
     - 端口：8000
     - 环境变量：参考`.env.example`
     - 数据库：使用CloudBase自带的PostgreSQL（需要申请）

3. **配置域名和HTTPS**
   - 在"路由配置"中绑定自定义域名
   - 开启HTTPS（CloudBase自动提供SSL证书）

### 2.2 前端部署（CloudBase静态托管）

1. **构建前端生产版本**
   ```bash
   cd C:\Users\Admin\Desktop\MolingProject\moling-web
   npm run build
   ```

2. **上传到CloudBase静态托管**
   - 进入"静态托管" -> "文件管理"
   - 上传`out/`目录下的所有文件
   - 配置索引文档为`index.html`
   - 配置错误文档为`404.html`

3. **配置API代理**
   - 在"设置" -> "域名管理"中配置反向代理
   - 将`/api/v1/*`代理到后端云服务

## 方案3：传统服务器部署

### 3.1 服务器准备

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装依赖
sudo apt install -y python3.12 nodejs npm postgresql redis nginx

# 创建部署用户
sudo useradd -m -s /bin/bash moling
sudo su - moling
```

### 3.2 后端部署

```bash
# 克隆代码
cd ~
git clone <your-repo-url> moling-server
cd moling-server

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
pip install gunicorn

# 配置环境变量
cp .env.example .env
nano .env  # 修改DATABASE_URL、REDIS_URL、LLM_API_KEY等

# 运行数据库迁移
python -m alembic upgrade head

# 使用Systemd管理服务
sudo nano /etc/systemd/system/moling-backend.service
```

**Systemd服务文件：**
```ini
[Unit]
Description=Moling Backend API
After=network.target postgresql.service redis.service

[Service]
Type=notify
User=moling
Group=moling
WorkingDirectory=/home/moling/moling-server
Environment="PATH=/home/moling/moling-server/venv/bin"
ExecStart=/home/moling/moling-server/venv/bin/gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable moling-backend
sudo systemctl start moling-backend
sudo systemctl status moling-backend
```

### 3.3 前端部署

```bash
# 克隆代码
cd ~
git clone <your-repo-url> moling-web
cd moling-web

# 安装依赖
npm install -g pnpm
pnpm install

# 配置环境变量
cp .env.example .env.local
nano .env.local  # 修改NEXT_PUBLIC_API_URL为后端域名

# 构建生产版本
pnpm run build

# 使用PM2管理进程
npm install -g pm2
pm2 start pnpm --name "moling-frontend" -- start
pm2 save
pm2 startup
```

### 3.4 Nginx反向代理配置

**文件：`/etc/nginx/sites-available/moling`**
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 后端API
    location /api/v1/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket支持（如果需要）
    location /ws/ {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/moling /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# 配置HTTPS（使用Let's Encrypt）
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 常见问题

### Q1: 数据库迁移失败
**A:** 确保PostgreSQL正在运行，且`DATABASE_URL`配置正确。手动运行迁移：
```bash
cd moling-server
python -m alembic upgrade head
```

### Q2: Redis连接失败
**A:** 检查Redis是否正在运行：
```bash
redis-cli ping  # 应该返回PONG
```
如果Redis可选，可以在`.env`中设置`REDIS_URL=`为空，应用会优雅降级。

### Q3: LLM API调用失败
**A:** 检查`LLM_API_KEY`和`LLM_API_BASE`配置是否正确。测试API连接：
```bash
curl -X POST "https://api.deepseek.com/v1/chat/completions" \
  -H "Authorization: Bearer $LLM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"deepseek-chat","messages":[{"role":"user","content":"Hello"}]}'
```

### Q4: 前端无法调用后端API
**A:** 检查：
1. `NEXT_PUBLIC_API_URL`是否配置正确
2. 后端CORS配置是否包含前端域名
3. Nginx反向代理配置是否正确

### 性能优化

1. **后端**：
   - 增加Gunicorn worker数量（`-w 4` -> `-w 8`）
   - 启用PostgreSQL连接池
   - 使用Redis缓存频繁查询

2. **前端**：
   - 启用Next.js增量静态再生（ISR）
   - 配置CDN加速静态资源
   - 优化图片（使用WebP格式）

3. **数据库**：
   - 创建索引（`CREATE INDEX ...`）
   - 定期清理旧数据（如生成的临时文件）
   - 使用pgvector索引加速向量查询
