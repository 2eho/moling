## 项目介绍

墨灵（Moling）是一个 AI 辅助小说创作平台，采用前后端分离架构：

| 组件 | 技术栈 | 说明 |
|------|--------|------|
| 前端 | Next.js 19 + React 19 + TypeScript | moling-web |
| 后端 | FastAPI + PostgreSQL + Redis | moling-server |
| 向量数据库 | PostgreSQL + pgvector | 知识库向量存储 |

### 部署架构（云服务器版）

```
用户浏览器
  → http://服务器IP:8080/moling/
       ↓
  Nginx（宿主机，端口 8080）
    /moling       → proxy_pass → moling-web:3000（前端）
    /moling/api/  → proxy_pass → moling-app:8000（后端）
       ↓
  Docker 容器：moling-web / moling-app / moling-redis
```

## 快速开始

### 1. 克隆项目

```bash
git clone <repository-url>
cd MolingProject
```

### 2. 配置环境变量

```bash
cp moling-server/.env.example moling-server/.env
vim moling-server/.env
```

**必需修改的配置：**

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `SECRET_KEY` | JWT 签名密钥 | `openssl rand -hex 32` 生成 |
| `POSTGRES_PASSWORD` | 数据库密码 | 强密码 |
| `REDIS_PASSWORD` | Redis 密码 | 强密码 |
| `LLM_API_KEY` | LLM API 密钥 | `sk-...` |

### 3. 构建并启动（本地开发）

```bash
# 构建镜像
docker build -t moling-app:latest moling-server/
docker build -t moling-web:latest moling-web/

# 启动所有服务
docker run -d --name moling-redis redis:7-alpine
docker run -d --name moling-app -p 8000:8000 --env-file moling-server/.env moling-app:latest
docker run -d --name moling-web -p 3000:3000 moling-web:latest

# 访问
open http://localhost:3000
open http://localhost:8000/docs
```

## 云服务器部署实战

> **实际案例**: 部署到 `124.222.163.79`，通过 `http://124.222.163.79:8080/moling/` 访问

### 部署步骤

#### 步骤 1：构建 Docker 镜像

```bash
# 登录服务器
ssh root@124.222.163.79

# 克隆代码
cd /opt
git clone <repository-url>
cd MolingProject

# 构建后端镜像（注意：moling-server/Dockerfile 已修复 libffi8 等问题）
cd moling-server
docker build -t moling-app:latest .

# 构建前端镜像
cd ../moling-web
docker build -t moling-web:latest .
```

**如果构建失败，参见[故障排除](#故障排除)。**

#### 步骤 2：启动 Docker 容器

```bash
# 创建专用网络
docker network create moling-network 2>/dev/null || true

# 启动 Redis
docker run -d --name moling-redis \
  --network moling-network \
  --restart unless-stopped \
  redis:7-alpine

# 启动后端（确保 .env 文件已配置）
docker run -d --name moling-app \
  --network moling-network \
  -p 8000:8000 \
  --env-file /opt/MolingProject/moling-server/.env \
  --restart unless-stopped \
  moling-app:latest

# 启动前端
docker run -d --name moling-web \
  --network moling-network \
  -p 3000:3000 \
  --restart unless-stopped \
  moling-web:latest

# 检查容器状态
docker ps
```

#### 步骤 3：配置 Nginx 反向代理

在宿主机上创建 `/etc/nginx/conf.d/moling.conf`：

```nginx
server {
    listen 8080;
    listen [::]:8080;
    server_name _;

    # 根路径重定向到 /moling
    location = / {
        return 301 /moling;
    }

    # 前端（Next.js standalone 模式，basePath: "/moling"）
    location /moling {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后端 API（去掉 /moling 前缀后转发）
    location /moling/api/ {
        rewrite ^/moling(/api/.*)$ $1 break;
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_connect_timeout 60s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
        proxy_buffering off;
    }

    # API 文档
    location /moling/api/docs {
        proxy_pass http://127.0.0.1:8000/api/docs;
        proxy_set_header Host $host;
    }
}
```

**应用配置：**

```bash
# 测试配置
sudo nginx -t

# 重载（不中断服务）
sudo nginx -s reload

# 如果 Nginx 未启动
sudo systemctl start nginx
sudo systemctl enable nginx
```

#### 步骤 4：验证访问

```bash
# 本地测试前端
curl -s http://127.0.0.1:3000 | head -20

# 本地测试后端
curl http://127.0.0.1:8000/api/health

# 外部访问测试
curl -I http://124.222.163.79:8080/moling/
```

浏览器访问：`http://124.222.163.79:8080/moling/`

### Next.js 子路径部署配置

`moling-web/next.config.ts` 必须配置 `basePath`：

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  basePath: "/moling",   // 所有前端路由自动加 /moling 前缀
  output: "standalone",  // Docker 中直接用 node server.js 启动
};

export default nextConfig;
```

配置后，Next.js 会自动将 `_next/static/...` 等静态资源路径加上 `/moling` 前缀，Nginx 无需额外 `rewrite`。

## Docker 镜像构建

### 后端 Dockerfile 要点（moling-server/Dockerfile）

```dockerfile
FROM python:3.11-slim

# Debian Trixie 用 libffi8（不是 libffi7）
RUN sed -i 's/deb.debian.org/mirrors.aliyun.com/g' /etc/apt/sources.list.d/debian.sources 2>/dev/null; \
    apt-get update && apt-get install -y --no-install-recommends \
    gcc libffi8 libpq5 curl

WORKDIR /app

# 先装依赖（此时 NODE_ENV 尚未设为 production）
COPY pyproject.toml .
RUN pip install --upgrade pip && \
    pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/ && \
    pip install --no-cache-dir .

COPY . .

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**关键修复：**
- `libffi7` → `libffi8`（Debian Trixie 包名变更）
- 添加 pip 阿里云镜像加速
- `pyproject.toml` 中 `bcrypt>=4.0,<5.0`（原 `bcrypt<4.1` 无可用版本）
- 添加 `email-validator>=2.2.0`（pydantic EmailStr 依赖）

### 前端 Dockerfile 要点（moling-web/Dockerfile）

```dockerfile
FROM node:22-alpine AS builder

WORKDIR /app

# 关键：先 npm ci（此时 devDependencies 会被安装），再设 NODE_ENV=production
COPY package*.json ./
RUN npm ci

ENV NODE_ENV=production

COPY . .
RUN npx next build

# standalone 模式输出到 .next/standalone
FROM node:22-alpine
WORKDIR /app
COPY --from=builder /app/.next/standalone ./
EXPOSE 3000
CMD ["node", "server.js"]
```

**关键修复：**
- `ENV NODE_ENV=production` 必须在 `npm ci` **之后**，否则 devDependencies（含 typescript）不会被安装
- 启动命令用 `node server.js`（standalone 模式）

## 故障排除

### 1. 端口已被占用

**现象**: `Bind for 0.0.0.0:8080 failed: port is already in use`

**解决**:

```bash
# 查看占用端口的进程
sudo lsof -i :8080

# 如果 8080 被占用，改用其他端口（如 8081）
# 修改 Nginx 配置中的 listen 指令
sudo vim /etc/nginx/conf.d/moling.conf
sudo nginx -t && sudo nginx -s reload
```

### 2. 数据库连接失败

**现象**: 后端启动后报错 `connection to server at "localhost" (::1), port 5432 failed`

**解决**: 检查 `.env` 中的数据库配置，确保 PostgreSQL 容器已启动且网络可达。

### 3. 前端无法访问后端 API

**现象**: 浏览器控制台报错 `Failed to load resource: net::ERR_CONNECTION_REFUSED`

**解决**: 检查 Nginx 配置，`/moling/api/` 的 `proxy_pass` 是否正确指向后端容器 IP。

### 4. 镜像构建失败（Debian Trixie 包名）

**现象**: `Unable to locate package libffi7`

**原因**: `python:3.11-slim` 基础镜像已升级到 Debian Trixie，`libffi` 包名变为 `libffi8`

**解决**: 修改 Dockerfile，将 `libffi7` 改为 `libffi8`

### 5. bcrypt 版本约束无效

**现象**: `Could not find a version that satisfies the requirement bcrypt<4.1`

**原因**: `pyproject.toml` 中 `bcrypt<4.1` 无下限，且最新 bcrypt 已 > 4.1

**解决**: 改为 `bcrypt>=4.0,<5.0`

### 6. NODE_ENV 顺序错误导致构建失败

**现象**: Next.js 构建时报错 `Cannot find module 'typescript'`

**原因**: `ENV NODE_ENV=production` 在 `npm ci` 之前，导致 devDependencies 被跳过

**解决**: 调整 Dockerfile 顺序，先 `npm ci` 再设 `NODE_ENV=production`

### 7. Windows 大小写不敏感导致 Linux 构建失败

**现象**: 本地能跑，Docker Linux 环境报 `Module not found: Can't resolve './Settings.module.css'`

**原因**: Windows 不区分大小写，`import './Settings.module.css'` 能匹配 `settings.module.css`，但 Linux 区分

**解决**: 确保 import 语句和文件名完全一致（注意大小写）

### 8. Next.js 动态路由 slug 名称冲突

**现象**: 构建时报错 `You cannot use different slug names for the same dynamic path`

**原因**: 同一路由目录下有多个动态路由文件夹，使用了不同的参数名（如 `[id]` 和 `[projectId]`）

**解决**: 统一使用一个参数名，删除重复的路由目录

### 9. pydantic 需要 email-validator

**现象**: 后端启动报错 `ImportError: email-validator is not installed`

**原因**: pydantic 的 `EmailStr` 类型需要 `email-validator` 包

**解决**: 在 `pyproject.toml` 中添加 `email-validator>=2.2.0`

### 10. Nginx 反向代理重定向循环

**现象**: 浏览器报错 `ERR_TOO_MANY_REDIRECTS`

**原因**: `location /moling/` 只匹配带末尾斜杠的路径，Next.js 重定向导致循环

**解决**: 改为 `location /moling`（去掉末尾斜杠）

### 11. Docker 镜像构建极慢（国内网络）

**现象**: `apt-get update` 或 `pip install` 需要 17 分钟以上

**解决**: 在 Dockerfile 中添加国内镜像源：

```dockerfile
# apt 使用阿里云镜像
RUN sed -i 's/deb.debian.org/mirrors.aliyun.com/g' /etc/apt/sources.list.d/debian.sources 2>/dev/null; \
    apt-get update && apt-get install -y --no-install-recommends ...

# pip 使用阿里云镜像
RUN pip install --upgrade pip && \
    pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/ && \
    pip install --no-cache-dir .
```

### 12. Next.js standalone 模式启动报错

**现象**: 容器启动后报错 `Cannot find module '/app/server.js'`

**原因**: `next.config.ts` 中未配置 `output: "standalone"`，或 Dockerfile 启动命令错误

**解决**:
- 确保 `next.config.ts` 中有 `output: "standalone"`
- Dockerfile 启动命令用 `CMD ["node", "server.js"]`
- 确保 `.next/standalone` 目录被正确复制到运行时镜像

### 13. 页面能访问但样式丢失

**现象**: 页面 HTML 能加载，但 CSS/JS 返回 404

**原因**: `basePath` 配置后，静态资源路径变为 `/moling/_next/static/...`，但 Nginx 配置不正确

**解决**: 确保 Nginx `location /moling` 正确代理到前端容器，Next.js standalone 模式会自动处理静态资源路径

## 生产环境建议

### 安全配置

1. **修改默认密码**: 所有默认密码必须通过 `openssl rand -base64 32` 生成
2. **启用 HTTPS**: 申请 SSL 证书，配置 Nginx 强制跳转 HTTPS
3. **配置防火墙**: 仅开放必要端口（22、80、443、8080）

### 性能优化

1. **数据库优化**: 调整 PostgreSQL 配置，增加共享内存
2. **Redis 优化**: 配置最大内存和淘汰策略
3. **后端多进程**: 使用 Gunicorn 或 uvloop 提升并发能力

### 监控和日志

1. **日志收集**: 使用 `docker logs` 或配置集中式日志
2. **错误追踪**: 配置 Sentry 或其他错误监控服务
3. **健康检查**: 配置 Docker 健康检查，自动重启异常容器
