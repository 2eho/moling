## 二、问题逐项分析与改进方案

### 🔴 问题 1：图标系统使用 emoji / Unicode 符号

**现状**: Sidebar 使用 📚 ✒️ 📖 🔔 ⚙️ 等 emoji；工作台 TopBar 使用 ☰ ✦ ⚙ 🛡 ← 等 Unicode

**问题**: 
- emoji 在不同操作系统上渲染差异巨大（Windows vs macOS vs iOS）
- 无法控制颜色，无法和品牌色统一
- 无法做 hover/active 状态
- 显得廉价、不专业

**改进方案**: 统一替换为 SVG 图标系统

```
├── src/components/icons/          # 新建
│   ├── index.ts                   # 统一导出
│   ├── ProjectIcon.tsx            # 项目（文档图标）
│   ├── WriteIcon.tsx              # 写作（笔图标）
│   ├── VaultIcon.tsx              # 四库（书架图标）
│   ├── BellIcon.tsx               # 通知
│   ├── SettingsIcon.tsx           # 设置
│   ├── ChevronLeftIcon.tsx        # 左箭头
│   ├── ...                        # 20-30 个常用图标
```

**SVG 图标优势**: 矢量缩放不失真、可设置 stroke/fill 颜色跟随 CSS var、hover 时可变色、打包后可 tree-shaking

### 🔴 问题 2：工作台 TopBar 按钮布局混乱

**现状**: TopBar 的按钮从左到右：
```
[☰] [←] [项目名 ▾] [章节选择器] | [🛡] [⚙] [用户头像] [✦]
```

**问题**:
- 功能按钮混杂：面板切换、返回、项目切换、章节选择、健康、设置、用户、另一个面板切换
- "返回"按钮在 Sidebar 之外再加一个，与 Sidebar 的"项目"导航功能重叠
- 健康状态用 🛡 emoji 占用重要位置
- 左右面板切换按钮放在 TopBar 两端，视觉不统一

**改进方案**:

```
├─ [Panel ←] [项目名] [▾] [Chapter Selector] │ [✨ 抽卡] [🔮 生成] │ [通知] [设置] [用户头像]
```

- 左侧：面板折叠 + 项目信息（主要导航）
- 中间/右侧分隔：核心操作（抽卡/生成）放到显眼位置
- 右侧：辅助功能（通知/设置/用户）

---

### 🔴 问题 3：LeftPanel 与 "四库" 页面功能重叠

**现状**: 
- LeftPanel 包含：人物、时间线、剧情承诺、世界观（正是"四库"内容）
- 同时存在独立的 `/vaults/[projectId]` 页面
- 两个地方展示相似内容，但实现方式不同

**问题**:
- 用户困惑：应该用 LeftPanel 还是去独立页面？
- 维护成本翻倍
- 编辑体验不一致

**改进方案**: 
- **LeftPanel** → 轻量级的"快速参考/预览"模式（只读摘要 + 链接到详情）
- **独立四库页面** → 全功能管理页面（增删改查 + 关联关系）
- 或者：LeftPanel 只展示当前章节相关的内容（上下文参考），完整管理去四库页面
- 增加从 LeftPanel 直接跳转到四库页面的"查看更多"链接

---

### 🔴 问题 4：ToolBar（底部工具栏）功能定义模糊

**现状**:
```
[🎴 抽卡] [✨ 生成] ← 左侧操作
[AI指令输入框...        ] ← 中间输入
[📥 收纳] [📤 拒稿] ← 右侧操作
```

**问题**:
- "抽卡"在 ToolBar 里，也在 RightPanel 的 Tab 里——功能重复
- "收纳/拒稿"命名不够直观（收纳 = 确认生成内容；拒稿 = 拒绝并重写）
- AI 指令输入框占据大量空间，但使用频率不高
- 整体功能层级不清晰

**改进方案**:
```
左侧操作区              │ 中间操作区              │ 右侧操作区
[抽卡 🎴] [生成 ✨]     │ [AI 指令... 输入框]      │ [✓ 确认] [↻ 重写]
```

- "收纳" → "确认 ✓" （更直观）
- "拒稿" → "重写 ↻" （更积极）
- 抽卡在主 ToolBar 保持，RightPanel 的抽卡 Tab 移除或改为"卡牌库"管理
- AI 指令输入框折叠成可展开模式

---

### 🔴 问题 5：Sidebar 功能定位不清晰

**现状**: 
- Sidebar 只在非工作台/非项目列表页面显示（settings, notifications, admin 等）
- 工作台有自己的 TopBar 导航
- 项目列表页既没有 Sidebar 也没有专用的导航

**问题**:
- 导航系统有 3 种不同的模式（有 Sidebar / 无 Sidebar + TopBar / 移动端 BottomNav）
- 用户在不同页面间跳转时，导航布局不一致，容易迷失
- Sidebar 内容太少（只有 4 个导航项），占用大量空间

**改进方案**: 
- **统一导航结构**: 所有登录后的页面使用一致的 Sidebar + TopBar 模式（类似 Linear/Notion）
- Sidebar 增加内容：最近项目列表、快捷操作、AI 助手入口
- 工作台的 TopBar 简化为"面包屑 + 项目名 + 关键操作"
- 或者：移除 Sidebar，改为统一的 TopBar 主导航（类似 GitHub/Discord）

---

### 🔴 问题 6：RightPanel 四个 Tab 信息密度过高

**现状**: 4 个 Tab（章节列表 / AI 建议 / 卡牌 / 生成历史）

**问题**:
- Tab 切换成本高，每次点击才能查看不同内容
- "章节列表"实际上在 RightPanel 放不下，需要滚动
- "生成历史"几乎没用（只有一个当前任务）
- 4 个 Tab 挤在狭窄的侧栏里，每个 Tab 的内容都很单薄

**改进方案**:
- **合并为 2 个核心 Tab**: "上下文"（章节列表 + AI 建议）和 "卡牌"（当前卡牌）
- 或者使用手风琴/折叠面板结构，同时展示多个内容块
- "生成历史"移到一个独立的轻量级 Modal 或 Page
- 或者参考 Notion 的右侧"评论/属性"面板——只展示当前上下文的辅助信息

---

### 🔴 问题 7：Landing Page 视觉丰富但缺乏层次

**现状**: 全屏 Hero → Stats → Features → Gacha → Testimonials → CTA → Footer，还有浮动装饰动画

**问题**:
- 内容太多，信息层级不够清晰
- Desktop 和 Mobile 是两套完全独立的渲染（代码重复严重）
- 浮动卡牌动画分散注意力
- 翻转卡牌的交互很有趣但实现复杂，容易出问题

**改进方案**:
- **精简内容**: Hero → Features → CTA（保留最核心的转化点）
- **统一响应式**: 使用 CSS Grid/Flexbox 自适应，不再写两套
- 浮动动画减弱，改为微妙的视差效果
- 翻转卡牌改为"点击展开详情"的 simpler 交互

---

### 🔴 问题 8：Auth 页面过于简单

**现状**: 居中卡片，Logo + 标题 + Tab 切换 + 表单

**问题**:
- 没有品牌故事传达
- 没有 Landing 到 Auth 的过渡感（从丰富到极简，落差大）
- 没有社交登录或其他登录方式的暗示

**改进方案**:
- 在左侧或背景中添加一些品牌视觉元素（如浮动卡牌、写作场景插画）
- 增加"特性亮点"展示（登录页面也是品牌传达窗口）
- 添加 Demo/体验入口（不登录也能先感受产品）

---

### 🔴 问题 9：Projects 页面布局可以更高效

**现状**: Header + Stats + Grid

**问题**:
- 统计卡片（Stats）占用大量首屏空间但信息少
- 搜索框和"新建"按钮在 Header 右侧，有足够空间但视觉上不平衡

**改进方案**:
- 将 Stats 整合到 Header 区域或 Grid 上方紧凑展示
- 搜索和新建按钮调整间距和视觉权重
- 列表/网格视图切换功能
- 按更新时间排序，增加最近编辑的时间显示

---

### 🔴 问题 10：Pricing 页面使用 alert()

**现状**: 错误和成功反馈使用 `alert()` 弹窗

**问题**: 这是最基本的前端反模式——alert() 无法自定义样式，阻断用户操作流

**改进方案**: 使用 Toast 组件替代所有 alert()

## 三、功能按钮位置重新规划

### 全局导航结构（统一方案）

```
┌─────────────────────────────────────────────────────┐
│                    TopBar                            │
│ [☰ Sidebar] [墨灵] [项目名 ▾] │ [💰 定价] [🔔] [设置] [头像] │
├─────────────┬───────────────────────────┬─────────────┤
│  Sidebar     │      Main Content          │  Context    │
│              │                           │  Panel      │
│ 📊 仪表盘    │                           │  (上下文    │
│ 📚 项目      │                           │   辅助)     │
│ ✍️ 写作中    │                           │             │
│ 📖 四库      │                           │             │
│ ├──────      │                           │             │
│ 💡 AI 助手   │                           │             │
│ ⚙️ 设置      │                           │             │
└─────────────┴───────────────────────────┴─────────────┘
```

### 工作台内功能分布

| 区域 | 功能 | 优先级 |
|------|------|:------:|
| **TopBar 左侧** | 面板折叠、项目名、章节选择器 | P0 |
| **TopBar 中部** | 抽卡 🎴、生成 ✨（核心操作） | P0 |
| **TopBar 右侧** | 通知、设置、用户、时钟 | P1 |
| **Left Panel** | 四库预览（角色/时间线/伏笔/世界观） | P0 |
| **Editor 区域** | 写作编辑区（核心内容） | P0 |
| **Right Panel** | 上下文信息（当前章节相关建议、参考） | P1 |
| **ToolBar 底部** | AI 指令、确认/重写、字数统计 | P1 |
| **浮动按钮** | 快速回到顶部、快捷抽卡 | P2 |

### 核心操作流

```
Step 1: 选择/创建章节（TopBar 左侧 或 RightPanel）
     ↓
Step 2: 抽卡（TopBar 中部 或 快捷键 Cmd+D）
     ↓
Step 3: 选择卡牌组合（Modal）
     ↓
Step 4: 生成内容（自动开始）
     ↓
Step 5: 查看结果（Editor 自动滚动）
     ↓
Step 6: 确认 ✓ 或 重写 ↻（ToolBar）
```
