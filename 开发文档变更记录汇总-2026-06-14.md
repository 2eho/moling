## 变更 1：LLM 配置管理 API 实现

### 实现位置
- **路由**: `app/router/admin.py`
- **配置管理**: `app/config.py`
- **Schema**: `app/schemas/admin.py`

### 新增 API 端点

#### GET /api/v1/admin/llm-config

**功能**: 获取当前 LLM 配置（API Key 脱敏显示）

**响应示例**:
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "api_base": "https://api.deepseek.com",
    "model": "deepseek-chat",
    "is_configured": true,
    "api_key_masked": "sk-ab12****wxyz"
  }
}
```

**实现逻辑**:
1. 优先读取数据库 `system_config` 表
2. 如果数据库无记录，读取环境变量
3. API Key 返回脱敏格式（前4位 + "****" + 后4位）

---

#### POST /api/v1/admin/llm-config

**功能**: 保存 LLM 配置到数据库，并立即更新运行时

**请求体**:
```json
{
  "api_key": "sk-xxxx",
  "api_base": "https://api.deepseek.com",
  "model": "deepseek-chat"
}
```

**实现逻辑**:
1. 保存到数据库 `system_config` 表（覆盖写入）
2. 调用 `set_override()` 更新内存中的配置覆盖
3. 下一次 LLM 调用将使用新配置（无需重启）

---

#### POST /api/v1/admin/llm-config/test

**功能**: 测试 LLM 连接（使用当前配置）

**实现逻辑**:
1. 使用当前生效配置（DB override > env var）
2. 发送一条测试消息 `"Hello, are you working?"`
3. 返回连接是否成功 + 响应时间

---

### 配置分层策略（已实现）

墨灵的配置从三个层级加载，**后一层级覆盖前一层级**：

```
Layer 3: 数据库配置（运行时动态）  ← 最高优先级
          ↓ 覆盖
Layer 2: 环境变量 / .env 文件
          ↓ 覆盖
Layer 1: 代码默认值                   ← 最低优先级
```

**关键函数**:
- `get_effective_llm_config()`: 获取生效配置（自动合并三层）
- `set_override(key, value)`: 设置运行时覆盖（立即生效）

## 变更 2：LLM 统一客户端实现

### 实现位置
- **客户端**: `app/llm/client.py`
- **依赖**: `httpx` (异步 HTTP) + `tenacity` (自动重试)

### 核心功能

#### 1. 支持流式和非流式调用

```python
client = LLMClient()

# 非流式
resp = await client.chat(
    messages=[{"role": "user", "content": "Hello"}],
    stream=False
)

# 流式
async for chunk in await client.chat_stream(
    messages=[{"role": "user", "content": "Hello"}],
    stream=True
):
    print(chunk)
```

#### 2. 自动重试（Tenacity）

**重试策略**:
- **触发条件**: `httpx.TimeoutException`, `httpx.ConnectError`, `httpx.RemoteProtocolError`
- **最大尝试次数**: 3 次
- **等待策略**: 指数退避（1s → 2s → 4s，最大 10s）
- **重试日志**: 自动记录每次重试（方便排查）

#### 3. 运行时配置重载

```python
class LLMClient:
    def _get_config(self) -> dict[str, str]:
        """每次调用时读取最新配置"""
        return get_effective_llm_config()
```

**设计理由**：
- 不在 `__init__` 时创建 `httpx.AsyncClient`（避免配置过时）
- 每次请求创建新的 `httpx.AsyncClient`（使用最新配置）

### 8.X 统一 LLM 客户端

墨灵实现了一个统一的 LLM 客户端 `LLMClient`，封装所有 LLM 调用细节。

#### 核心功能

| 功能 | 实现方式 |
|------|---------|
| 流式/非流式调用 | `chat()` / `chat_stream()` |
| 自动重试 | `tenacity` 库，指数退避 |
| 运行时配置重载 | 每次调用读取 `get_effective_llm_config()` |
| Token 计数 | 近似估算（基于字符数） |

#### 使用示例

**非流式调用**:
```python
from app.llm.client import LLMClient

client = LLMClient()

resp = await client.chat(
    messages=[
        {"role": "system", "content": "你是一个小说创作助手"},
        {"role": "user", "content": "请生成第 1 章开头"}
    ],
    model="deepseek-chat",
    temperature=0.7,
    max_tokens=4096
)
# resp = {"choices": [{"message": {"content": "..."}}], "usage": {...}}
```

**流式调用**:
```python
async for chunk in await client.chat_stream(
    messages=[...],
    model="deepseek-chat"
):
    # chunk = "生成的内容片段"
    print(chunk)
```

#### 重试策略

使用 `tenacity` 实现自动重试：

```python
_RETRY_DECORATOR = retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=1, max=10),
    retry=retry_if_exception_type(
        (httpx.TimeoutException, httpx.ConnectError, httpx.RemoteProtocolError)
    ),
    before_sleep=before_sleep_log(logger, logging.INFO),
    reraise=True,
)
```

**重试场景**：
- LLM API 超时（Timeout）
- 网络连接错误（ConnectError）
- 远程协议错误（RemoteProtocolError）

**不重试场景**：
- 4xx 错误（参数错误，重试无用）
- 认证失败（API Key 错误）

```

这意味着：
1. 管理员通过后台页面修改配置 → 保存到数据库
2. 调用 `set_override()` 更新内存
3. 下一次 LLM 调用自动使用新配置（无需重启）

## 变更 3：风格指纹系统实现

### 实现位置
- **分析器**: `novel_dissector/core/style_analyzer.py`
- **集成位置**: `app/genre/a5_profile_output.py` (作为 Genre Profile 的 `style_fingerprint` 字段)

### 功能概述

风格指纹系统用于**量化分析小说文风**，提取可复现的文风特征，用于：
1. **拆书引擎**：分析热门书的文风 → 作为 Genre Profile 的一部分
2. **冷启动**：新书创建时，加载同类型 Genre Profile 的文风特征 → 指导 AI 生成相似文风
3. **用户导入**：分析用户已有内容的文风 → 保持后续生成的一致性

---

### 实现架构

#### Step 1 维度（零依赖 · 纯规则）

**文件**: `novel_dissector/core/style_analyzer.py`

**7 个文风维度**:

| 维度 | 说明 | 计算方法 |
|------|------|----------|
| `sentence_len` | 平均句长 | `总字符数 / 句子数` |
| `para_len` | 平均段落长度 | `总句子数 / 段落数` |
| `dialogue_ratio` | 对话占比 | `对话行数 / 总行数` |
| `complex_word_ratio` | 复杂词占比 | `≥3字词组数 / 总词数` |
| `punct_density` | 标点密度 | `标点符号数 / 总字符数` |
| `passive_ratio` | 被动语态占比 | `被动句数 / 总句子数` |
| `tone` | 语调分类 | 规则判断 → `"严肃" | "轻松" | "庄重" | ...` |

**输出示例**:
```json
{
  "sentence_len": 42.5,
  "para_len": 3.8,
  "dialogue_ratio": 0.35,
  "complex_word_ratio": 0.18,
  "punct_density": 0.12,
  "passive_ratio": 0.08,
  "tone": "严肃"
}
```

#### Step 2 维度（可选 · LLM 辅助）

**3 个高阶维度**（需要 LLM 理解）:

| 维度 | 说明 |
|------|------|
| `narrative_distance` | 叙事距离（近景第一人称 / 远景第三人称） |
| `emotional_intensity` | 情感强度（0-10 评分） |
| `rhythm_pattern` | 节奏模式（紧凑 / 舒缓 / 多变） |
