核心改进建议 · 按优先级排序

从产品定位出发，独立判断每个功能按钮的最佳位置

1
工作台顶栏重设计
现在
[☰] [←] [项目名▾] [章节▾] ··· [🛡] [⚙] [头像▾] [✦] — 8个元素平铺
建议
左侧组：面包屑(我的作品 / 项目名) + 章节选择器
右侧组：[参考面板切换] [AI面板切换] + 分割线 + [头像下拉(含设置/退出)]
移除：返回按钮(面包屑已承载)、健康图标(用Toast)、设置图标(收入下拉)
理由
遵循"频率分层"原则：高频操作（章节切换、面板切换）在顶栏；低频操作（设置、退出）收入下拉菜单
2
底部工具栏重新组织
现在
ToolBar 只有抽卡按钮，信息密度低
建议
左：当前章节名 + 实时字数统计
右：[预览] [抽卡] [AI生文]
三按钮递进：预览 → 抽卡 → 生文，符合创作流程
理由
底部栏是创作流程的"操作中枢"，抽卡→生文是最核心的路径，必须在此处而不是分散在别处
3
导航栏标准化
现在
桌面端：工作台/项目页无侧边栏。移动端：底部导航只有4个tab
建议
桌面端：保留当前全宽布局，顶栏增加全局导航入口(项目列表、四库、导入、设置)
移动端：底部导航增加到5个：[项目] [工作台] [四库] [导入] [设置]
理由
工作台是高频页面，全宽布局是对的。但需要在顶栏提供"快速跳转到其他模块"的入口
4
着陆页增强
现在
Hero → Features → Gacha Preview → Stats → CTA → Footer。结构合理但缺少定价入口。
建议
1. 导航添加"定价"链接
2. 增加 Pricing Section 预览卡片
3. 尾部 CTA 旁加"查看定价"次要按钮
5
项目卡片信息增强
现在
卡片点击弹出 Detail Modal，多一步操作
建议
卡片直接显示关键信息(章节数、字数、最后更新时间)，点击直接进入工作台。右上角 ··· 菜单可编辑/删除。减少 Modal 操作。
理由
用户来项目列表的目的是"进入某个项目继续写"，不是"查看项目详情"。直接进入是最短路径。
6
后台管理 Tab 式布局
现在
三个 Tab 组件(Overview/Users/LlmConfig)，缺少支付记录
建议
5 个 Tab：概览 / 用户管理 / LLM用量 / 支付记录 / 配置。顶部 Tab bar + 指标卡片 + 数据表格的经典后台布局。

移动端策略

当前移动端已有很好的手势操作和抽屉面板。以下是增强建议：

📱
底部导航 5 Tab
项目 | 工作台 | 四库 | 导入 | 设置
每个 Tab 图标 + 文字，当前活跃态用 indigo 高亮。5个Tab比4个更完整覆盖核心功能。
改进
👆
保留手势操作
当前的手势系统很好：左滑→参考面板、右滑→AI面板、下滑→专注模式。这是移动端核心竞争力之一，保留并强化。
已有
⚡
快速操作 FAB
在工作台移动端，右下角增加浮动操作按钮(FAB)：抽卡。这是最高频操作，应该随时可触达，不需要打开底部工具栏。
新增

改动总览

优先级 页面/组件 改动内容 预期效果
P0 工作台顶栏 功能分组：导航组(面包屑+章节) / 面板组(参考+AI切换) / 用户组(头像下拉) 信息清晰度↑ 操作效率↑
P0 底部工具栏 左：章节名+字数 / 右：预览→抽卡→生成三按钮递进 创作流程更流畅
P1 全局导航 移动端底部5Tab(项目/工作台/四库/导入/设置)；桌面端顶栏快捷跳转 跨模块跳转效率↑
P1 着陆页 增加定价入口链接、增强视觉层次 转化率↑
P2 项目卡片 卡片直接显示章节/字数/时间，点击直接进入工作台 减少操作步骤
P2 后台管理 5 Tab 经典后台布局，增加支付记录和LLM用量 管理效率↑
P2 移动端 5 Tab 底部导航 + 抽卡 FAB + 手势保留 移动端体验完整
