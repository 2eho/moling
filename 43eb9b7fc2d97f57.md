## §3 共享布局模式

> 以下 3 种布局模式覆盖所有页面。Agent 需根据页面类型选择对应模式。

### 模式 A：全屏单行（Landing）

```
┌─────────────────────────────────────┐
│  [NavBar 固定]                        │
├─────────────────────────────────────┤
│  [Hero Section]                      │  可滚动
│  [Features Cards]                    │  全屏高度
│  [抽卡展示]                          │
│  [Footer]                            │
└─────────────────────────────────────┘
```

**特点**：body 可滚动、无固定侧栏、无固定底栏（Mobile 除外）。

### 模式 B：三栏工具（Workspace）

```
┌──────┬──────────────────────┬───────┐
│ 左侧  │      中栏             │ 右侧  │  ← 三栏固定，body overflow hidden
│ 面板  │      工作区           │ 面板  │     每栏各自 scroll
│ 280px │     flex: 1          │ 240px │
└──────┴──────────────────────┴───────┘
```

**特点**：`overflow: hidden` 在 body，各栏单独 `overflow-y: auto`。

### 模式 C：内容卡片（Auth / Pricing / 404）

```
┌─────────────────────────────────────┐
│                                     │
│         ┌─────────────────┐         │
│         │  卡片内容居中      │         │
│         │  (max-width ~400px)│        │
│         └─────────────────┘         │
│                                     │
└─────────────────────────────────────┘
```

**特点**：body 可滚动、内容居中对齐、max-width 限制。

### 模式 D：Shell 路由（Projects / NewProject / Vaults / Settings / Notifications / Import）

```
┌─────────────────────────────────────┐
│  [Header Bar · 52px]                 │
├─────────────────────────────────────┤
│  [Scrollable Content Area]           │  内容区域可滚动
│    ┌──────────┐                      │  padding 左右 16-32px
│    │ 内容区域   │                      │  max-width: 880px 居中（Desktop）
│    │ max-w 880 │                      │
│    └──────────┘                      │
│                                     │
├─────────────────────────────────────┤
│  [Mobile Bottom Nav · 56px]         │  ← 仅 Mobile 显示
└─────────────────────────────────────┘
```

## §5 Dual-Shell 响应式架构

### 5.1 架构模式

墨灵 7 个主要页面使用 **Dual-Shell** 响应式架构：

```
CSS: Mobile Shell 默认显示（dom 优先），Desktop Shell 在 ≥769px 显示
HTML: 两套完整的 DOM 子树，各自独立
JS: 通过 classList 切换控制两棵子树的显示状态
```

**断点**：
| 断点 | 范围 | 显示 Shell |
|------|------|-----------|
| Mobile | < 769px | `#mobile-shell` / `.mobile-shell` |
| Desktop | ≥ 769px | `#desktop-shell` / `.desktop-shell` |

### 5.2 桌面端的 Shell 内部降级

≤900px 时，Desktop Shell 内部的 grid 布局自动降级：
```
2列grid → 1列
3列grid → 1列
4列grid → 2列
main-scroll padding: 32px → 16px
```

### 5.3 Shell 显示控制 CSS

```css
/* Mobile default */
.mobile-shell { display: block; }
.desktop-shell { display: none; }

@media (min-width: 769px) {
  .mobile-shell { display: none; }
  .desktop-shell { display: block; }
}
```

### 5.4 哪些页面使用 Dual-Shell

| 页面 | 使用 Dual-Shell? | Shell 类型 | 备注 |
|------|-----------------|-----------|------|
| Landing | ✅ | `.mobile-shell` + `.desktop-shell` | 两个完全独立的 DOM 子树 |
| Projects | ✅ | `#mobile-shell` + `#desktop-shell` | Desktop 直接渲染、Mobile 用 JS show/hide |
| NewProject | ✅ | `#mobile-shell` + `#desktop-shell` | 同上 |
| Vaults | ✅ | `.mobile-shell` + `.desktop-shell` | 用 media query + JS show/hide |
| Settings | ✅ | `.mobile-shell` + `.desktop-shell` | 仅 media query 控制 |
| Workspace | ✅ (混合) | 单一 DOM + 局部 media query | Desktop fixed 三栏 + Mobile 切换为 stack |
| Import | ✅ | `.mobile-shell` + `.desktop-shell` | 完整的 Dual Shell |
| Auth | ❌ | 单一 Shell | 居中卡片布局 |
| Pricing | ❌ | 单一 Shell | 768px 断点折叠导航 |
| Notifications | ❌ | 单一 Shell | 480px 微调 |
| 404 | ❌ | 单一 Shell | 全屏居中 |

### 6.3 状态机

#### Desktop NavBar

| 状态 | 触发条件 | 视觉变化 |
|------|---------|---------|
| 顶部（默认） | `scrollY ≤ 20` | `background: transparent`，无 blur，无 shadow |
| 滚动后 | `scrollY > 20` | `background: rgba(13,15,26,0.85)` + `backdrop-filter: blur(16px)` + `box-shadow` |
| Nav 链接 hover | hover | `::after` width `0 → 100%`（amber 下划线动画 200ms） |
| Nav CTA hover | hover | `translateY(-1px)` + `box-shadow: 0 4px 12px rgba(212,168,67,0.3)` |

#### Desktop Feature 卡片

| 状态 | 触发条件 | 延迟 |
|------|---------|------|
| 隐藏（初始） | 进入视窗前 | `opacity: 0; translateY(24px)` |
| 可见 | IntersectionObserver 触发，添加 `.visible` | 0s / 0.12s / 0.24s（3 张 card 依次） |
| hover | 鼠标悬停 | `translateY(-4px)` + border-color 变亮 + `--shadow-lg` |

#### Desktop Gacha 卡牌

| 状态 | 触发条件 | 视觉变化 |
|------|---------|---------|
| 背面（默认） | 初始 | 显示花纹边框 + "灵感"文字 |
| 翻面 | click / Enter / Space | 内层 `rotateY(180deg)`，显示正面（稀有度渐变边框+内容） |
| Legendary 脉冲 | 始终（仅 legendary） | `box-shadow` 在 `0 0 12px` 和 `0 0 24px` 之间循环（2s） |

#### Mobile Feature Slider

| 状态 | 触发条件 | 视觉变化 |
|------|---------|---------|
| 默认（第1张） | 初始 | 第1个 dot `.active`（width 20px, brand-indigo） |
| 滑动中 | `.m-features-slider` scroll | debounced(80ms) 计算索引，更新 dot 的 `.active` |
| Card active | touch/click | `transform: scale(.98)` |

#### Mobile Flip Card

| 状态 | 触发条件 | 视觉变化 |
|------|---------|---------|
| 正面（默认） | 初始 | 显示卡牌正面 |
| 翻面 | click | `transform: rotateY(180deg)`（0.6s cubic-bezier 过渡） |

#### Scroll Fade In（Mobile + Desktop）

| 状态 | 触发条件 | 变化 |
|------|---------|------|
| 隐藏 | 进入视窗前 | `opacity: 0; translateY(20px)` |
| 可见 | IntersectionObserver（threshold 0.15） | `opacity: 0→1; translateY(20px)→0`（0.6-0.8s） |

### 6.4 动画细节

| @keyframes | 作用对象 | 时长 | 循环 | 说明 |
|-----------|---------|------|------|------|
| `dFloatCard` | 6 个浮动画（Desktop Hero背景） | 18-25s 不等 | infinite | translateY + rotate，延迟错开 -5s 到 -15s |
| `dPulse` | Hero 标签圆点 | 2s | infinite | opacity 1 ↔ 0.4 |
| `dShimmer` | Hero 高亮文字 | 6s | infinite （paused） | 渐变位移，默认暂停，不运行 |
| `dCtaGlow` | CTA 按钮 `::before` 伪元素 | 2s | infinite | 白色高光从左到右扫过（hover 时触发） |

| `dLegendaryGlow` | Legendary 卡牌边框 | 2s | infinite alternate | box-shadow 脉冲效果 |

### 7.2 表单状态机

| 状态 | 触发条件 | 显示内容 | API Endpoint |
|------|---------|---------|-------------|
| 登录（默认） | 初始 / 从重置返回 | 邮箱 + 密码 + 忘记密码? + 登录按钮 | `POST /api/v1/auth/login` |
| 注册 | 点击"注册"Tab | 笔名 + 邮箱 + 密码 + 确认密码 + 勾选协议 + 注册按钮 | `POST /api/v1/auth/register` |

| 设置新密码 | URL 带 `?reset=<token>` | 新密码 + 确认密码 + 重置密码按钮 | `POST /api/v1/auth/password-reset` |

### 7.3 数据模型

```typescript
interface AuthFormData {
  email: string;
  password: string;
  nickname?: string;       // 注册时
  confirmPassword?: string; // 注册/设置密码时
  agreed?: boolean;        // 注册时
}

interface LoginResponse {
  access_token: string;
  refresh_token: string;
  user: { id: string; nickname: string; email: string; };
}

interface RegisterResponse {
  id: string; email: string; nickname: string; created_at: string;
}

interface PageState {
  activeTab: 'login' | 'register';
  activeForm: 'login' | 'register' | 'reset' | 'resetDone' | 'setPassword';
  resetToken: string | null;
}
```

### 7.4 交互事件表

| 触发器 | 动作 |
|--------|------|
| Tab 按钮 click | 切换 `.active` class，切换显示对应表单 |
| 密码切换 click | toggle input type `password` ↔ `text`，按钮文字"显示"↔"隐藏" |
| 登录表单 submit | 验证非空 → loading spinner → POST API → 成功: localStorage 存 token + 800ms 后跳转 `/projects` → 失败: toast 错误 |
| 注册表单 submit | 验证 (笔名+邮箱+密码≥6+确认一致+勾选协议) → loading → POST → 成功: toast + 切回登录tab+预填邮箱 → 失败: toast |
| 忘记密码 submit | 验证邮箱 → loading → POST → 成功: 显示"重置已发送"视图 → 失败: toast |
| 设置密码 submit | 验证(密码≥6+一致+token存在) → POST → 成功: toast + 1.5s 后回登录页 |
| 忘记密码? 链接 | 隐藏 tabs + 所有表单 → 显示忘记密码表单，预填登录邮箱 |
| ← 返回登录 | 隐藏重置表单 → 显示 tabs + 登录表单 |
| 页面加载 IIFE | 检测 URL `?reset=<token>` → 设置隐藏 token → 隐藏 tabs → 显示设置密码表单 |
| Toast 自动隐藏 | 3s 后自动关闭 |

### A.3 常见错误

| 错误 | 正确做法 |
|------|---------|
| 直接硬编码颜色 `#6366f1` | 使用 `var(--color-brand-indigo)` |
| 文字/背景对比度不足 | 主文字用 primary(4.5:1)，次要文字用 secondary(3.5:1) |
| Mobile 可交互元素 < 44px | 所有按钮/输入框至少 32px，推荐 44px |
| 忘记 `-webkit-font-smoothing` | 必须加 `antialiased` |
| 动画时间太短 < 200ms | hover 反馈 100ms，面板切换 300ms，进入动画 500ms |
| Dual-Shell 忘了同步 | 每次状态切换同时更新 desktop 和 mobile DOM |
| 省略 loading/empty/error 态 | 每个数据展示组件都要处理这 3 种状态 |
