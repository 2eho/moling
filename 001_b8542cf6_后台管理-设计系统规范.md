## 1. 设计系统匹配与推荐

### 候选方案对比

| 方案 | 设计系统 | 匹配度 | 特征 | 适合原因 |
|------|---------|--------|------|---------|
| **A ★ 首选** | **MoLing Dark UI（扩展）** | ★★★★★ | 深色主题 `#0d0f1a`，indigo+amber 品牌色，Noto Serif SC / PingFang SC 字体体系 | 已有完整设计系统，后台作为产品线自然延伸，保持品牌一致性，降低开发成本 |
| B | Stripe Dashboard | ★★★★☆ | 深色管理后台标杆，极致的数据表格/统计卡片/支付流水 UI，清晰的信息层级 | 金融级的数据展示参考，支付记录模块与其高度契合，表格/统计卡片设计可直接借鉴 |
| C | Vercel Dashboard | ★★★☆☆ | 简洁数据面板风格，高效的响应式布局，干净的信息密度控制 | 用户管理/用量趋势图区域的布局与信息密度可参考，但风格偏浅色系需适配深色主题 |

### 最终推荐：方案 A — MoLing Dark UI（扩展）

**理由**：

1. **品牌一致性**：MoLing 已有完整的深色设计系统和 11 个页面的设计遗产，后台管理作为第 12 个页面应无缝融入
2. **令牌复用**：可直接复用现有 `--color-bg`、`--color-surface`、`--color-brand-indigo` 等核心令牌，仅需扩展后台专用的语义色、数据展示色等
3. **字体体系延续**：Noto Serif SC（标题）+ PingFang SC（正文/UI）+ JetBrains Mono（数据/数字）的字体栈完美适配后台数据展示场景
4. **组件继承**：Button 系统（7 种变体）、Tag 系统（6 种）、Card、Toast、ProgressBar 等直接可用
5. **风格借鉴**：在具体组件实现上可参考 Stripe Dashboard 的表格设计（行高亮、排序指示器、空状态）和 Vercel Dashboard 的布局网格（StatCard 网格、内容区比例）

## 9. Agent Prompt Guide

### 9.1 设计与开发注意事项

1. **色彩一致性**：后台管理页面的所有色彩须使用本文档定义的 CSS 变量，不可硬编码色值。主色参考 `--color-brand-indigo`，数据展示色用 `--chart-*` 系列。

2. **字体体系**：
   - 后台页面大标题使用 `--font-heading`（Noto Serif SC）
   - 统计大数字、表格数据使用 `--font-mono`（JetBrains Mono）
   - 正文/UI 文字使用 `--font-body`（PingFang SC）
   - 严格遵循字号对照表，不可混用

3. **组件复用优先**：优先使用主系统已有组件（Button、Tag、Card 等），仅当主系统组件不满足后台需求时才使用本规范的新组件。

4. **WCAG 对比度**：
   - 所有文字颜色与背景的对比度 ≥ 4.5:1（AA 标准）
   - 大字号文字（≥18px bold 或 ≥24px）对比度 ≥ 3:1
   - `--color-text-disabled` (`#3d4166`) 仅在禁用态使用，不可用于正文

5. **深色主题适配**：
   - 后台管理在深色背景上展示大量数据，注意信息密度的平衡
   - 表格交替行色 `--color-table-row-alt` 与背景的差异应微妙（仅 1-2% 亮度差）
   - 图表配色使用 `--chart-*` 系列，确保在深色背景上具有足够的饱和度

6. **空状态处理**：每个数据区域（表格/图表/活动列表）必须处理空状态：
   - 表格：显示空状态插画 + "暂无数据" + 建议操作文字
   - 图表：显示占位图 + "暂无数据" 
   - 活动列表：显示"暂无最近活动"

7. **加载状态**：所有数据请求区域必须处理加载态：
   - 表格：6 行骨架屏脉冲
   - StatCard：数值区域骨架屏，图标保持显示
   - 图表：图表区域脉冲占位

8. **错误状态**：API 请求失败时：
   - 表格：保留表头 + 错误提示行 + "重试"按钮
   - 统计卡片：数值显示 `--` + 错误提示 tooltip
   - 图表：显示"加载失败，点击重试"

9. **动效原则**：
   - 反馈类动画（hover/click）使用 `--duration-fast`（100ms）
   - 布局过渡（侧边栏折叠）使用 `--duration-slow`（300ms）
   - 弹窗进入使用 `--ease-spring` 弹性曲线
   - 严格遵循 `prefers-reduced-motion` 媒体查询

10. **侧边栏规范**：
    - 展开态：240px，显示图标 + 文字标签
    - 折叠态：64px，仅图标居中，hover 显示 tooltip
    - 移动端：抽屉式（从左侧滑入），遮罩层 backdrop
    - 激活项左侧 3px indigo 指示器，不可使用其他颜色

11. **表格规范**：
    - 表头固定（sticky），内容区滚动
    - 最小表格宽度 700px（支持横向滚动）
    - 排序指示器使用 unicode 箭头（↑↓），激活态为 indigo 色
    - 操作列固定最右，使用图标按钮以节省空间

12. **Tab 切换**：
    - 点击 Tab 切换内容面板，带动画（fadeIn + translateY 4px）
    - URL hash 同步（可选）
    - Tab 激活态：底部 2px indigo 色条 + 文字变 indigo
