# 墨灵 · 前后端接口映射文档

> 将 7 个 HTML UI 原型与后端 API 规范对齐，每个交互点标注对应的端点、请求格式、响应处理方式。
> **目标**：前端开发者拿到这个文档 + HTML原型 = 可以直接写出完整的 SPA。

---

## 技术栈决策

| 层 | 选型 | 版本 | 原因 |
|:--|:-----|:----:|:-----|
| **前端框架** | React + Next.js | 19 + 15.5.x (App Router) | AI 最擅长 React+TS，生成代码质量最高；476K Stack Overflow 问题 = 遇到坑 AI 能修 |
| **语言** | TypeScript | ≥5.5 | 全栈统一语言，AI 最熟练的类型体系 |
| **路由** | Next.js App Router | 内置 | 8 个页面一一映射为文件路由，layout 嵌套支持双壳设计 |
| **样式** | CSS Modules | — | 8 个 HTML 原型的 CSS 直接提取复用，不需要 CSS-in-JS |
| **状态管理** | React Context + useReducer | 内置 | 项目初期 8 个页面共享状态有限（用户/当前项目/Phase4 状态），Context 够用。复杂后加 Zustand |
| **API 请求** | fetch（浏览器原生）| — | 71 个端点全是 REST，不需要 axios 等额外依赖 |
| **部署** | Vercel / Docker | — | Next.js 在 Vercel 上零配置部署；也可 docker build 自托管 |

> 完整对比过程见对话记录：2026-06-11。核心决策链：Svelte(性能好但招不到人+AI不会) → Vue(中文好但AI生成质量低) → React(AI最熟、TS最好、生态最全)。

## 九、全局规则

### 9.1 认证要求

所有 `/api/v1/projects/*` 路径需要 `Authorization: Bearer <token>` header。
401 响应 → 前端跳转到登录页。

### 9.2 分页

列表类接口（章节列表、角色列表、卡牌池）支持 `?page=&page_size=` 参数。
默认 `page_size=20`。

### 9.3 错误处理

```javascript
// 前端统一错误处理
async function apiCall(url, options) {
  try {
    const res = await fetch(url, { headers: { 'Authorization': `Bearer ${getToken()}` }, ...options });
    if (res.status === 401) { redirectToLogin(); return; }
    if (res.status === 403) { showToast('无权访问'); return; }
    if (res.status === 404) { showToast('资源不存在'); return; }
    if (res.status === 429) { showToast('操作太频繁，请稍后重试'); return; }
    if (!res.ok) { const err = await res.json(); showToast(err.message); return; }
    return await res.json();
  } catch (e) {
    showToast('网络错误，请检查连接');
  }
}
```

### 9.4 数据联动

#### 确认收纳 → Phase 4 异步 → 下一章

```
用户点"确认收纳"
  POST /chapters/:id/confirm → {task_id}
  按钮显示 "收纳中🔄"
  
  // 异步轮询
  setInterval(() => {
    GET /generate/:task_id/status → {status}
    if status === "done"   → 按钮 "已收纳✅"
    if status === "failed" → 按钮 "收纳失败❌ 重试"
  }, 2000)
  
  // 用户点"下一章"
  POST /chapters/:id/generate → 生成时后端自动使用 is_draft=FALSE 的动态层
```
